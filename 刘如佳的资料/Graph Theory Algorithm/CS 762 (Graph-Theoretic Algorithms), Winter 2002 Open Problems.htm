Open

On this page is a list of open problems. Some may truly be open, for some I may simply not have found the right reference. New open problems will appear irregularly (whenever I stumble over one).

Any open problem that you solve will earn you some extra credit (how much depends on the problem). It als may be a good foundation for future research projects.

If you think you have a solution to something, contact me. If you think you have a solution to a closely related problem, let me know that, too.

  1. Maximum cut in interval graphs
    What is the status of the problem of finding a Maximum Cut in an interval graph? Is this NP-hard? Is this poly-time solvable? (It is known to be NP-hard in chordal graphs.)

  2. Interval graphs and Matroids
    Various greedy-algorithms work on interval graphs (e.g. for coloring, independent set.) Is there any connection between interval graphs and matroids that would explain why this is so?

  3. Hamiltonian cycle in chordal graphs
    What is the status of the problem of finding a Hamiltonian cycle in a chordal graph? Is this NP-hard? Is this poly-time solvable? How about perfect graphs?

  4. Intersection graphs with few elements
    We know that every graph is an intersection graph if we represent each vertex by Sv={all edges incident to v}. With this formulation, we need m different elements (one for each edge) to create the sets for the vertices.
    Can we do this with n elements? Or maybe even less? Or can we show that m elements are needed for some graphs?

  5. t-interval graphs
    Every graph is a Delta-interval graph. Is this the best possible, i.e., is there a graph that isn't a (Delta-1)-interval graph, for arbitrary Delta? (Note that for Delta=2, the 4-cycle is a 2-interval graph but not a 1-interval graph.)

  6. SP-graphs as boxicity-2 graphs
    Is it true that every (simple) SP-graph is a boxicity-2 graph? (We know this for triangle-free 2-terminal SP-graphs. Dealing with cutvertices is easy, but how to add the triangles?)

  7. Recognizing 2-outer-planar graphs
    We can recognize a 2-outer-planar graph in O(n2). Can we do better? In particular, can we do O(n)?
